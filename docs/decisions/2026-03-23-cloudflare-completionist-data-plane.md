# 2026-03-23 Cloudflare Completionist Data Plane

## Status

Accepted

## Context

Skyviz remains a browser-local static product, but completionist mode needs a hot shared flight snapshot that refreshes every `5` minutes.

The old approach refreshed that snapshot by republishing the GitHub Pages site through GitHub Actions. That made live-data freshness depend on Actions queueing, cache warmth, and Pages deploy latency, even though the static shell itself had not changed.

The repository also proved easy to point at the wrong Cloudflare account when Wrangler auth exposed multiple memberships. That is an operational risk because queue, bucket, workflow, and worker writes are account-scoped.

## Decision

Use a Cloudflare-native completionist data plane:

- `Cron Trigger` starts the refresh cadence.
- `Workflow` owns one run per schedule slot.
- `Queue` fans out tile fetch jobs.
- `Durable Object` coordinates run state, dedupe, publish readiness, and single-writer publish.
- `R2` stores per-tile artifacts plus published manifests and snapshots.
- the browser reads the production completionist manifest from `site/data/runtime-config.json`
- local preview and offline validation still use `site/data/live/` fixtures generated by `scripts/refresh_completionist_snapshot.py`

Lock all Cloudflare operations in this repository to:

- email: `seansailer28@gmail.com`
- account id: `172da47da00e3b33810d2e9c73c9a0b9`

The worker configuration is the source of truth for that account lock, and manual deploy/provision flows must verify Wrangler auth before any write command.

## Consequences

- Completionist refreshes no longer require republishing the GitHub Pages shell.
- The last good live manifest can stay published even if a later run fails.
- Queue retries, tile splitting, and publish coordination are handled as pipeline concerns instead of deploy concerns.
- GitHub Pages now only ships the static shell plus slower-moving generated artifacts.
- Local development keeps a fixture path and does not require live Cloudflare access to exercise completionist mode.
- The repo now needs explicit operational guardrails so Cloudflare writes never target the wrong account.
